# Запросы

> Центр управления заявками клиентов. Отслеживайте статус выполнения запросов, контролируйте сроки и приоритизируйте задачи.

## Обзор

Запросы (или тикеты) — это структурированные задачи, возникающие в процессе общения с клиентом. В отличие от чата, который представляет собой поток сообщений, запрос имеет четкую цель, статус выполнения, приоритет и дедлайн.

## Запрос vs Чат

- **Чат**: Место для общения, уточнения деталей, быстрых вопросов.
- **Запрос**: Единица работы, которую нужно выполнить (например, "Подготовить отчет за 3 квартал", "Сверить акт").

## Основные функции

### Создание нового запроса

Запрос можно создать двумя способами:

1. **Из чата**: Прямо в процессе разговора с клиентом, чтобы зафиксировать задачу.
2. **Вручную**: Через кнопку "Новый запрос" в разделе Запросы.

При создании укажите:

- Тему запроса.
- Клиента.
- Описание задачи.
- Приоритет (Низкий, Средний, Высокий).
- Крайний срок (Дедлайн).

### Статусы запросов

Статус отражает текущий этап жизненного цикла заявки:

- **Новый**: Заявка создана, но работа по ней еще не начата.
- **В работе**: Оператор занимается решением задачи.
- **Ожидает ответа клиента**: Нужна дополнительная информация от клиента. Таймер SLA может быть приостановлен.
- **Выполнен**: Работа завершена, решение предоставлено.
- **Отменён**: Заявка неактуальна или создана ошибочно.

### Приоритеты и SLA

Приоритет влияет на допустимое время реакции и решения:

- **Высокий**: Требует самой быстрой реакции.
- **Средний**: Стандартные задачи.
- **Низкий**: Задачи, которые могут подождать.

### Фильтрация и сортировка

Для эффективной работы с очередью задач используйте фильтры:

- По статусу (например, показать только "В работе").
- По приоритету (сначала важные).
- По ответственному сотруднику.
- По клиенту.

### Массовые действия

В списке запросов можно выбрать несколько строк галочками и применить действие ко всем сразу:

- Изменить статус.
- Назначить ответственного.
- Удалить (если есть права).

### Связь запроса с чатом

Каждый запрос может быть связан с чатом.

- В карточке запроса есть ссылка "Перейти в чат".
- В чате отображается плашка активного запроса.
  Это позволяет быстро восстановить контекст задачи, прочитав переписку.

## Частые вопросы

**Q: Можно ли переоткрыть выполненный запрос?**
A: Да, если клиент вернулся с уточнениями по той же теме, статус можно изменить обратно на "В работе".

**Q: Как изменить дедлайн запроса?**
A: Дедлайн рассчитывается автоматически на основе SLA, но менеджеры могут изменить его вручную в карточке запроса, если есть обоснование.
