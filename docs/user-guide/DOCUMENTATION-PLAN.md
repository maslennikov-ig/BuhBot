# План создания пользовательской документации BuhBot

> **Цель**: Создать полное руководство пользователя для платформы BuhBot
> **Целевая аудитория**: Операторы, менеджеры и администраторы бухгалтерских фирм
> **Язык**: Русский

### 2. dashboard.md (Дашборд)

**Путь**: `docs/user-guide/dashboard.md`

**Что описать**:
- Назначение дашборда — быстрый обзор текущей ситуации
- Виджеты и их значения:
  - **SLA Compliance** — процент соблюдения SLA, что означают цвета
  - **Время ответа** — средний показатель, тренд (стрелка вверх/вниз)
  - **Нарушения сегодня** — счётчик, сравнение со вчера
  - **Активные алерты** — критические/предупреждения/информационные
  - **Последние запросы** — таблица с фильтрацией
- Как интерпретировать метрики
- Рекомендуемые действия при отклонениях

**Роли**: Все пользователи

**Источник данных**: `frontend/src/app/dashboard/dashboard-content.tsx`

### 5. alerts.md (Алерты)

**Путь**: `docs/user-guide/alerts.md`

**Что описать**:
- Типы алертов:
  - **Критические** (красные) — требуют немедленного внимания
  - **Предупреждения** (жёлтые) — риск нарушения
  - **Информационные** (синие) — для сведения

  - Приближение дедлайна SLA

- Действия с алертами:
  - Просмотр деталей
  - Переход к связанному чату/запросу

- Настройка уведомлений

## Стиль и форматирование

### Общие требования

1. **Язык**: Русский, официально-деловой стиль, но дружелюбный
2. **Обращение**: На «вы» (с маленькой буквы)
3. **Формат**: Markdown с поддержкой GitHub
4. **Длина**: 300-800 слов на документ

### Структура каждого документа

```markdown
# Название раздела

> Краткое описание (1-2 предложения)

## Обзор

Общее описание функционала

## Основные функции

### Функция 1
Описание...

### Функция 2
Описание...

## Пошаговые инструкции

### Как сделать X

1. Шаг первый
2. Шаг второй
3. Шаг третий

## Частые вопросы

**Q: Вопрос?**
A: Ответ.

## См. также

- [Связанный раздел](./related.md)
```

### Визуальные элементы

- Использовать таблицы для сравнений и списков параметров
- Использовать блоки `> **Важно**:` для предупреждений
- Использовать `> **Совет**:` для полезных советов
- Код и названия кнопок выделять `обратными кавычками`
